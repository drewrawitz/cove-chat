# Attention

This context separates passive awareness, deliberate delivery, and an explicit request to respond. It keeps activity recoverable without treating every update as an interruption.

## Attention choices

**Follow**:
A workspace identity's passive interest in a topic, keeping its activity available without placing every message in the inbox.
_Avoid_: Subscribe to notifications, watch for alerts

**Notification**:
The deliberate delivery of one message to a recipient's workspace inbox. A notification does not imply urgency or external push delivery.
_Avoid_: Alert, push notification, unread activity

**Everyone Involved**:
A notification audience containing the topic author and every workspace identity that has authored a message in the topic. Reading, following, or receiving an earlier notification does not make someone involved.
_Avoid_: Everyone who viewed, topic followers

**Everyone in Channel**:
A notification audience containing every current channel member, not every workspace identity able to discover the channel.
_Avoid_: Everyone in workspace

**Response Request**:
An explicit request for a named recipient to respond to a topic. It remains pending until the recipient marks it handled.
_Avoid_: Mention, assignment, urgent notification

**Inbox**:
A workspace-scoped view of deliberately delivered messages, direct-topic activity, and pending response requests.
_Avoid_: Global inbox, unread counter

## Availability signals

**Presence**:
An ephemeral indication that a workspace identity is currently viewing a topic. It does not claim that the person is available or owes a response.
_Avoid_: Online status, last active, read receipt

**Away Status**:
A workspace-specific, time-bounded declaration that a person is unavailable, with an optional reason. The absence of an away status makes no claim about current availability.
_Avoid_: Working status, online status, global status
